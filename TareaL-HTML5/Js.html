Historia de JavaScript! El único lenguaje que entienden los navegadores!!

JavaScript es un lenguaje de programación de alto nivel que se utiliza principalmente para agregar interactividad a las páginas web. Fue creado por Brendan Eich y lanzado por primera vez en diciembre de 1995, se introdujo como una forma de agregar programas a páginas web en el navegador Netscape Navigator.

En el año 1994 aparece Netscape, el primer navegador masivo de la historia a partir de Mosaic, netscape no tenía ingresos ni modelo de negocios, pero al momento de salir a la bolsa como era la siguiente revolución de la tecnología ganó muchísimo dinero en su salida a la bolsa. Ahí fue donde inicio lo que conocemos como la la guerra de los navegadores ya que Bill Gates(Microsoft) desarrolló su propio navegador web llamado Internet Explorer que atacó con todo, esta tenía recursos prácticamente ilimitados.

Netscape se dió cuenta de que las páginas web son bastante básicas, no había interacción, solamente estaban para consumir como una revista pero en una pantalla, por lo que decidieron crear un lenguaje de programación que funcionara en el navegador. Un lenguaje básico de scripting(la escritura y ejecución de scripts o programas de secuencias de comandos). Es decir, un lenguaje de comandos específicos para hacer algunas cosas básicas.

Brendan Eich fue quien creo JavaScript en apenas una semana porque estaban en plena guerra de los navegadores. Eich hizo lo que podía en una semana, sin embargo este tuvo errores y arrastró la mala fama de JavaScript durante varios años, al inicio este se llamaba LiveScript, pero por la fama de Java en ese momento le cambiaron el nombre a JavaScript.

Pese a que Netscape hizo su estandarización en ecma(que obviamente ignoro Microsoft), en 1999 salio la última versión de su estándar. De 1999 hasta el año 2009(10 años), JavaScript no tuvo una nueva especificación, este fue abandonado por una década.

El renacimiento!

En el año 2002 aparece un libro que marca una especie de antes y después en el mundo de JavaScript, llamado JavaScript: The Good Parts, de Douglas Crockford. Este libro básicamente te dice que JavaScript no es un mal lenguaje, sino que tú eres un mal programador. Este libro realmente le devolvió la vida a JavaScript porque le dijo a la gente: oye, tú lo estás usando mal. JavaScript es excelente, fíjate las cosas que se pueden hacer con él.

En 2002 también se lanza Firefox, un navegador que competía contra Internet Explorer en una segunda guerra de los navegadores (la primera la perdió Netscape :c ). Netscape fue arrasado de la faz de la tierra por Microsoft, pero en 1998 Netscape abre su código y con eso, se crea el concepto de Open Source.

Microsoft jugaba sucio.

Microsoft tuvo un juicio de monopolio porque internet estaba instalado en Windows y no podías desinstalarlo, tampoco podías cambiar de navegador predeterminado. O sea, habían jugado bastante sucio esta guerra de navegadores.

Internet Explorer llegó a tener más del 95% del mercado de Internet. Incluso, la gente creía que la “e” de Internet Explorer era realmente Internet.

Frameworks JavaScript.

Como lenguaje de programación la evolución de JavaScript ha sido enorme y sigue creciendo mediante nuevas implementaciones, frameworks y librerías que se utilizan para diferentes usos. Todo esto hace imposible para los desarrolladores poder conocer todo de este lenguaje. Es aquí donde los frameworks se presentan como una especie de navaja suiza para facilitar su trabajo.

Estos son los más populares:

  • jQuery.js-2006: uno de los clásicos, han pasado muchos años desde que apareció en el mercado y se convirtió en el convirtió en el amo y señor absoluto de la web por una década.

    La explosión de jQuery permitía que algunas tareas, que se mandaban al servidor, ahora se les encargaran al navegador. Y como los navegadores eran más potentes, cada vez se desarrollaba más el concepto de aplicaciones web. Ya no sitios web, sino aplicaciones completas, con toda su lógica y negocio, pero funcionando en un navegador

  • Node.js-2009: Node.js no es un framework, es un entorno de ejecución, que son cosas totalmente diferentes. Con Node.js, ahora tenemos JavaScript en el servidor, en el navegador y podemos crear las famosas aplicaciones universales, que tengan JavaScript en todo el stack..
  • Angular.js-2010: este framework es desarrollado por Google y por tanto uno de los más populares. En la práctica no necesita presentación al ser el framework MVW(Model View WhatEver) que se ha encargado de traer orden a las aplicaciones JavaScript y potenciar las Arquitecturas SPA.
  • React.js-2013: la librería de Facebook orientada a la gestión de Interfaces de usuario. Muy extendido a la hora de desarrollar aplicaciones móviles. Este fue creado con la filosofía de crear componentes reutilizables, crear un virtual DOM, que era el DOM en memoria para no sobrecargar el navegador y un nuevo lenguaje, llamado JSX, que te permite escribir JavaScript como si estuvieras escribiendo HTML
  • Meteor.js: uno de los frameworks JavaScript que está adquiriendo mayor tracción en el mercado. Pensado para desarrollar aplicaciones JavaScript puedan ejecutar su código en entornos cliente cliente y servidor.

  • ECMAScript 2018 (ES9)

Ahora JavaScript se ha convertido en un lenguaje fundamental para el desarrollo web y se utiliza en una variedad de aplicaciones, desde páginas web simples hasta aplicaciones web complejas y móviles.
